HTML'de Dosya Yolları


Bir HTML belgesine herhangi bir dosya eklerken (örneğin resim) bu dosyanın yolunu yani bulunduğu konumu belirtmek gerekir. Aşağıdaki tabloda farklı yol belirtme yöntemlerini görebilirsiniz:

Yol Açıklama
"img src="resim.jpg" Resim, ekli olduğu HTML belgesi ile aynı klasörde
img src="resimler/resim1.jpg" resim1.jpg adlı dosya, ekli olduğu HTML belgesinin bulunduğu klasörün içinde bulunan Resimler adlı klasörün içinde
img src="/resim1.jpg" resim1.jpg sitenin ana klasöründe
img src="../resim1.jpg" resim1.jpg, ekli olduğu HTML belgesinin bir üst klasöründe


Peki HTML'de Dosya Yolları Nasıl Tanımlanır?

Bir dosya yolu, bir web sitesinde dosyanın bulunduğu konumu ifade eder. JavaScript dosyaları, CSS dosyaları, Resimler, Başka bir HTML dosyası gibi dosyalara link verirken dosya yollarını doğru belirtmeliyiz. Akti takdirde dosyalara verilen linkler hatalı olduğunda dosya bulunamaz ve dosya işlevini yerine getiremez.


Mutlak Dosya Yolları

Mutlak dosya yolu, dosya konumunu belirleyen tam URL yapısıdır.


Relative (Göreceli) Dosya Yolları

Göreceli dosya yolunda, dosyanın konumu ekli olduğu dosyaya göre belirlenir. Örneğin bir HTML dosyasında resim var. Göreceli dosya yolu belirtmede, dosya yolu dosyanın içinde olduğu HTML dosyasına göre verilir.


En İyi Yöntem Hangisi?

Bir dosyanın yolu belirlenirken mükün olduğunda göreceli dosya yolu kullanın. Bu şekilde hem dosya yolları daha kısa olacak hem de dosyanın yolu alan adınıza bağlı kalmayacaktır. Örneğin yerel bilgisayarınızda (web sitesini geliştirdiğiniz bilgisayarda) dosya yollarını düzeltmeniz gerekmeyecektir. Bilgisayarda oluşturduğunuz dosyaları sunucuya attığınızda herhangi bir değişiklik yapmadan çalışacaktır.
